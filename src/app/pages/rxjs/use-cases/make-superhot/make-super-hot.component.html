If we consider that streams needs subscribers to run...

we can undo that behaviour by making it like a promise:

Here we wrap up a stream in makeSuperHotObs$ to make it run like a promise

This can be useful for when logic is an observable but people may not subscribe or care about results

E.g. selecting a new tab, opening a dropdown, loading an image, etc

In code I call this a "super hot" observable, because not only does it share it's results like a hot observable but it runs eagerly like a promise
